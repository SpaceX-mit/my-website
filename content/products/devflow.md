# DevFlow Platform

> 面向中小团队的一站式开发效能平台

## 产品概述

DevFlow 是一个集代码管理、CI/CD、监控告警和团队协作于一体的开发效能平台，专为 10-100 人规模的技术团队设计。

## 核心功能

### 可视化流水线编排

通过拖拽方式构建 CI/CD 流水线，无需编写复杂的 YAML 配置：

- 内置 50+ 常用构建步骤
- 支持条件分支和并行执行
- 一键回滚到任意历史版本

### 多云部署支持

统一管理多个云环境的部署：

```yaml
# 一份配置，多云部署
deploy:
  targets:
    - cloud: aws
      region: us-east-1
      cluster: prod
    - cloud: aliyun
      region: cn-hangzhou
      cluster: prod-cn
```

### 实时监控大盘

内置监控能力，无需额外配置：

- 服务健康状态
- 部署成功率
- 响应时间趋势
- 错误率告警

### 团队权限管理

细粒度的权限控制：

| 角色 | 代码 | 部署 | 配置 | 管理 |
|------|------|------|------|------|
| 开发者 | 读写 | 测试环境 | 只读 | ✗ |
| 负责人 | 读写 | 全部 | 读写 | ✗ |
| 管理员 | 读写 | 全部 | 读写 | ✓ |

## 快速上手

### 1. 创建项目

连接你的 Git 仓库，DevFlow 自动检测项目类型并生成初始配置。

### 2. 配置流水线

```yaml
pipeline:
  stages:
    - name: 构建
      steps:
        - run: npm install
        - run: npm run build
        - run: npm test

    - name: 镜像打包
      steps:
        - docker: build
          tag: ${GIT_SHA}

    - name: 部署
      steps:
        - deploy: kubernetes
          manifest: k8s/
```

### 3. 设置告警

在 Web UI 中配置告警规则，支持 Slack、钉钉、企业微信等通知渠道。
